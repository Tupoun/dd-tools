SQL JOIN — přehled typů

Oracle DB / ANSI SQL  ·  příklady: account & client

Základní typy

INNER JOIN nejčastější

Vrátí pouze záznamy se shodou v obou tabulkách. Řádky bez shody se zahodí.

FROM A JOIN B ON A.id = B.id
příklad
FROM account a JOIN client c ON a.id = c.acc_id -- jen účty, které mají klienta
LEFT JOIN

Všechny záznamy z levé tabulky (A). Z pravé jen shoda — jinak NULL.

FROM A LEFT JOIN B ON A.id = B.id
příklad
FROM account a LEFT JOIN client c ON a.id = c.acc_id -- všechny účty, i bez klienta (NULL)
RIGHT JOIN

Všechny záznamy z pravé tabulky (B). Z levé jen shoda — jinak NULL.

FROM A RIGHT JOIN B ON A.id = B.id
příklad
FROM account a RIGHT JOIN client c ON a.id = c.acc_id -- všichni klienti, i bez účtu (NULL)
FULL JOIN = FULL OUTER JOIN

Vše z obou tabulek. Kde chybí vazba, doplní NULL na příslušné straně.

FROM A FULL JOIN B ON A.id = B.id
příklad
FROM account a FULL JOIN client c ON a.id = c.acc_id -- vše: účty bez klienta + klienti bez účtu

Left ↔ Right — ekvivalence

RIGHT JOIN lze vždy přepsat jako LEFT JOIN — stačí prohodit pořadí tabulek. V praxi se RIGHT JOIN skoro nepoužívá.

RIGHT JOIN
FROM account a RIGHT JOIN client c ON a.id = c.acc_id -- všichni klienti, i bez účtu
ekvivalentní LEFT JOIN
FROM client c LEFT JOIN account a ON a.id = c.acc_id -- stejný výsledek

Slovo OUTER je volitelné: LEFT OUTER JOIN = LEFT JOIN  ·  FULL OUTER JOIN = FULL JOIN

Ostatní

CROSS JOIN pozor — objem dat

Kartézský součin — každý řádek A s každým řádkem B. Bez ON podmínky. Výsledek: n × m řádků.

FROM A CROSS JOIN B
SELF JOIN aliasy nutné

Tabulka joinována sama se sebou. Typicky pro hierarchie (nadřízený–podřízený).

FROM emp e JOIN emp m ON e.mgr_id = m.id
UNION / UNION ALL není JOIN

Spojuje výsledky dvou SELECT dotazů vertikálně (přidává řádky, ne sloupce). Musí mít stejný počet a typ sloupců. UNION ALL ponechá duplicity.

SELECT id, name FROM account UNION SELECT id, name FROM archive_account
